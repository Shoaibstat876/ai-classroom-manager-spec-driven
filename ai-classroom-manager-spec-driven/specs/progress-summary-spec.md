# Progress Summary Module — Detailed Specification

## 1. Module Purpose
Generate short, clear, and helpful progress summaries about a student's learning performance or behaviour.  
The summaries must stay positive, constructive, and teacher-friendly.

---

## 2. Inputs

The module accepts:

- **Subject**  
- **Strengths**  
- **Areas for Improvement**  
- **Teacher Notes** (optional)  
- **Behaviour Notes** (optional)  
- **Recommendations** (optional)

---

## 3. Output Format (Strict Markdown)

The summary must follow this exact structure:

```markdown
# Progress Summary — [Subject]

## Strengths
- Point 1  
- Point 2  
- Point 3  

---

## Areas for Improvement
- Point 1  
- Point 2  

---

## Teacher Recommendations
- Recommendation 1  
- Recommendation 2  

---

## Additional Notes (Optional)
- Any extra comments
